# SignalForge Product Docs

## When To Use

Use this skill for product notes, design-partner discovery framing, insight synthesis definitions, demo narratives, and portfolio-facing documentation.

## Source Of Truth

Docs should define the intended product behavior before implementation. Keep product terms consistent across README, future specs, API contracts, UI copy, and demo materials.

## Product Boundaries

Stay focused on startup/product discovery and design-partner insight synthesis. Do not introduce industrial maintenance, equipment troubleshooting, manuals, CMMS, technician copilot, diagnostics, field service, PlantBrain, or Tenzin concepts.

## Docs To Add Later

- product brief
- insight schema reference
- demo scenario notes
- API contract notes
- AI extraction behavior notes
- interview demo script

## Tone

Write in a practical, concise, product-minded voice. Favor clear examples and decision rationale over broad strategy language.
